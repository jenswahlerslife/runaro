Map Debug Instructions

Follow these steps to debug the map rendering:

1. Check Database

Open your browser to the map page: http://localhost:8080

Log in and navigate to the map page

2. Open Dev Tools

Press F12 and go to the Console tab

Look for these debug messages:

  • [Map] Fetching activities for user: - Shows user ID
  • [Map] Fetched activities: - Shows how many activities were found
  • [Map] Raw activities data: - Shows the full database response
  • [Territory] Processing activity: - Shows each activity being processed
  • [Territory] Polyline length: - Shows if polyline data exists
  • [Map] Final result: - Shows how many territories were created
  • [Map] Rendering territory: - Shows each territory being drawn

3. Common Issues to Check

  • No activities found: Check if transfer from Strava success page worked
  • Activities found but no polylines: Check if Strava activities had GPS data
  • Polylines exist but no territories created: Check if polylines are valid
  • Territories created but not visible: Check bounds fitting and coordinate order
